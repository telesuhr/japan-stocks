## 2戦略の違い

| 項目 | lme_on_copper | topix_overnight |
|------|--------------|----------------|
| **シグナル** | 「今日」のLMEが東京時間に+1%動いたか | 「昨日から今朝」のTOPIXが+0.3%上がったか |
| **判断タイミング** | 引け直前15:25 (当日一日中監視が必要) | 朝9:05 (その日の行動計画が早く立てられる) |
| **発動頻度** | 少ない (月1-2回) | 多い (月4-5回) |
| **エッジの強さ** | 高 (Sharpe +11.5) | 中 (Sharpe +4.8) |
| **エッジの根拠** | LME銅のモメンタムが日本株に伝搬 | 夜間先物ギャップが翌日も継続 |

## 同日発動時のルール

両シグナルが同日に発動した場合:
- **lme_on_copper を優先**（Sharpeが高い）
- topix_overnight 側は当日キャンセル
- ポジションは¥5,000万（ダブルベットしない）

## 戦略追加ワークフロー

新戦略を採用する場合の手順:

1. `analyses/` で検証完了 (Sharpe ≥ 2.0 & N ≥ 30 & t-stat ≥ 2.0)
2. `strategies/<name>/` ディレクトリを作成
3. 最低限のファイルを揃える:
   - `README.md` — 戦略概要・バックテスト結果・オペレーション
   - `RULES.md` — 意思決定フローチャート
   - `DAILY_CHECKLIST.md` — 日次チェックリスト
   - `signal_check.py` — 発動判定スクリプト
   - `trade_log_template.csv` — 記録テンプレート
4. 本ファイルの戦略一覧に追記
