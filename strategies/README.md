## 戦略の追加ワークフロー

新しい戦略を採用する場合、以下の手順で本フォルダに追加する:

1. `analyses/` で検証完了 (Sharpe ≥ 2.0 & N ≥ 30 & t ≥ 2.0 の合格基準クリア)
2. 本フォルダ直下に `<strategy_name>/` ディレクトリを作成
3. 以下のファイルを最低限揃える:
   - `README.md` — 戦略全体概要
   - `DAILY_CHECKLIST.md` — 日次オペレーション
   - `RULES.md` — 意思決定ルール
   - `signal_check.py` — 発動判定スクリプト
   - `trade_log_template.csv` — トレード記録テンプレート
4. 本ファイルの「採用戦略一覧」に追記

## ポートフォリオ合成方針

同時採用する場合の資金配分ルール:

- **Core戦略 (lme_core5)**: 発動日は最優先、全額 ¥5,000万 を充てる
- **Overlay戦略**: Core 不発の日に発動。同じ¥5,000万を流用可
- **Satellite戦略**: Core と重複可能だが、総エクスポージャ¥5,000万を超えない範囲
- **上限**: 同時建玉 ¥5,000万 を守る (1銘柄の最大比率40%以内)
