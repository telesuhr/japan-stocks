# 意思決定フローチャート — Pre-Earnings Drift Long 戦略

---

## 🟢 Day N 15:25 — 翌日エントリー判定

```
START (Day N 15:25 — 引け直前)
  │
  ▼
① signal_check.py を実行
   - earnings_calendar から 3/5営業日後の決算予定を取得
   - 当日 (T) の終値・売買代金を取得
  │
  ▼
② シグナル発生銘柄 ≥ 1 か?
  │
  ├── No  → SKIP (no_signal)
  │
  └── Yes ▼

③ 戦略一時停止フラグが立っているか?
  │
  ├── Yes → SKIP
  │
  └── No  ▼

④ 現在の保有銘柄数 < 15 か?
  │
  ├── No  → SKIP (枠満杯)
  │
  └── Yes ▼

⑤ シグナル銘柄に既保有銘柄が含まれるか?
  │
  ├── Yes → 該当銘柄を除外
  │
  └── No  ▼

⑥ Day N+1 / N+2 に FOMC / 日銀会合 が控えているか?
  │
  ├── Yes → サイズ半減 または SKIP
  │
  └── No  ▼

⑦ シグナル銘柄に保有期間中の決算ピーク日 (5/14, 11/14等) が
   含まれる場合、ピーク日前のExit可能か?
  │
  ├── No  → 該当銘柄除外 (流動性悪化リスク)
  │
  └── Yes ▼

⑧ 売買代金降順で上位N銘柄を採択
   (N = min(15 - 現在保有数, シグナル銘柄数))
  │
  ▼
GO (採択銘柄を Day N+1 09:00 寄成 Long で発注)
```

---

## 🟡 Day N+1 09:00 — 寄成発注

```
START (Day N+1 09:00)
  │
  ▼
① 採択銘柄ごとに寄成 Buy 発注
  │
  ▼
② 寄付ストップ高 or 制限値幅 ±10% 超過?
  │
  ├── Yes → 該当銘柄キャンセル
  │
  └── No  ▼

③ 約定確認
  │
  ▼
④ trade_log.csv に約定価格・Exit予定日を記録
   (Exit予定日 = 決算予定日の前営業日)
```

---

## 🔴 決算前日 (T-1) 15:30 — 引成決済

```
START (各銘柄の Exit予定日 15:25)
  │
  ▼
① 翌営業日 (= 決算予定日) に決算予定があることを再確認
   ※発表延期の可能性をチェック
  │
  ├── 延期 → 翌日に持ち越し、新Exit予定日設定
  │
  └── 予定通り ▼

② 15:30 引成 Sell で全ポジション決済
  │
  ▼
③ 約定確認
  │
  ▼
④ trade_log.csv に決済価格・PnL を記録
```

**注**: SL/TP は設定しない。バックテストで「SLなし」が最良 (Sharpe +2.07) と確認済み。
        ただし、**-10% 以上のドローダウン**が出た場合のみ手動判断で早期Exit可。

## ⚠️ 戦略停止条件

以下のいずれかで戦略を一時停止:

1. **直近6ヶ月のSharpe < 0.5**
2. **5トレード連続で-3%以上の損失**
3. **日銀の大幅金融政策変更**
4. **暴落相場** (TOPIX 月次 -10% 以上)
5. **決算スケジュールの大幅変更** (取引所システム障害等)

停止解除条件:
- ホールドアウト12ヶ月で Sharpe > 1.5 を確認
- 戦略仮説の再検証完了

## 📋 他戦略との連携

### earnings_pead との"連続Hold"
同じ銘柄が PED → PEAD の連続シグナルを出す場合:
- T-5: PED で Long エントリー
- T-1: PED で Exit (発表前日引け)
- T:   決算発表 (引け後)
- T+1: ギャップ判定。+7%超なら PEAD で再エントリー
- T+6: PEAD で Exit (5営業日後)

→ **9営業日連続でエッジが取れる**

### bank_absorption との関係
銀行株の決算前後で両戦略が同時シグナルを出す可能性:
- 優先順位: **pre_earnings_drift > bank_absorption** (Sharpe比較)
- 既に PED で保有中 → bank_absorption はスキップ
- bank_absorption で保有中 → PED は新規シグナルとして追加可

### 他戦略
- vwap_morning_meanrevert / orb_breakout_long / eneos_vwap_trend: イントラ戦略のため独立
- lasertec_ma25_support: レーザーテック決算前後で重複可能性 → サイズ調整推奨

## 📅 例外処理

### ケース1: 寄付がストップ高
- 寄付直後にストップ高張り付き → スキップ
- 場中ストップ高解除されて売買代金が立てば、その時の値で再エントリー可

### ケース2: 保有期間中に決算が前倒し発表される
- 即時成行 Exit (発表後の急変動を回避)

### ケース3: 保有期間中に業績修正発表
- 上方修正 → 通常通り保有継続 (好材料)
- 下方修正 → 即時成行 Exit (悪材料)

### ケース4: 保有期間中の配当落ち日
- 通常通り保有継続 (PnLに含めず)

### ケース5: 上場廃止・MBO・TOB
- 発表があれば翌営業日成行 Exit

- [ ] **ピーク期のパフォーマンスを単独で評価**

- [ ] シグナル発生は月数件のみ
- [ ] 資金は他戦略に回す
