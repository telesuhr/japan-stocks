# Pre-Earnings Drift (PED) Long 戦略

**型**: スイング Long (3〜5営業日保有)
**バスケット**: プライム全銘柄 (除外: 医薬品 / 陸運業)
**昇格元**: [`analyses/20260512_pre_earnings_drift_validation/`](../../analyses/20260512_pre_earnings_drift_validation/)

---

## 1. 戦略の核

### コンセプト
**Pre-Earnings Drift (PED)** = 決算発表"前"のドリフト現象。
本決算・四半期決算の発表が近づくにつれて、期待感・事前買い・大手機関のポジショニングにより株価が上昇する傾向がある。

### 経済的根拠
- **期待感プレミアム**: 決算発表直前は注目度が高まり買いが先行
- **アナリスト予想集約**: 発表前にアナリストが予想を更新、それを見た投資家の事前買い
- **機関投資家のポジショニング**: 決算前に予想根拠で先回り買い
- **「earnings season rally」**: 米国市場でも同様のアノマリー実証済み

### シグナル
**Day N 15:25** に earnings_calendar をチェックし、以下を満たす銘柄を抽出:
- **本決算**: 5営業日後 (T+5) に予定 → 翌日エントリー、決算前日 (T+4) Exit (4日保有)
- **2Q/3Q決算**: 3営業日後 (T+3) に予定 → 翌日エントリー、決算前日 (T+2) Exit (2日保有)
- 当日売買代金 ≥ **5億円**
- 除外セクター (医薬品 / 陸運業) に該当しない
- プライム市場銘柄のみ

**1Q決算は除外** (バックテストSharpe < 0.6)

### エントリー / 出口
| 項目 | 内容 |
|------|------|
| エントリー | Day N+1 09:00 **寄成 Long** |
| 出口 | 決算前日 (T-1) 15:30 引成 Sell |
| **損切 (SL)** | **なし** (バックテストで悪化を確認) |
| 利食 (TP) | **なし** |
| ポジションサイズ | ¥1,000,000 / 銘柄 |
| 同時保有数上限 | **15銘柄** (決算ピーク期は満杯) |

## 6. 運用ルール

### 日次フロー
1. **15:25** Day N 引け直前
   `python3 signal_check.py` 実行
2. シグナル判定:
   - GO → Day N+1 09:00 寄成 Long で発注 (売買代金降順で上限15銘柄)
   - SKIP → ノートレード
3. **同時に**: 各保有銘柄の「決算前日」確認、該当銘柄を 15:30 引成 Sell

### 重複時のルール
- 既保有銘柄が同条件で再度シグナル発生 → **スキップ** (重複建玉禁止)
- 同時保有 15銘柄に達した場合、新規シグナルはスキップ

### 月次メンテナンス
- 直近1ヶ月の実績集計
- net mean > 0、勝率 > 50% を維持確認
- earnings_calendar データの欠損確認

### 半年次レビュー
- 24ヶ月バックテストを再実行
- 除外セクター見直し (新たに Sharpe<0 のセクターがあれば追加)
- doc_type 除外見直し

## 7. 想定資金枠

- 通常運用: ¥15,000,000 (15銘柄 × ¥100万)
- **ピーク期 (5月・8月・11月)**: 同時シグナル 15銘柄満杯
  - 5月: 本決算 (3月期決算) のピーク = 1日500銘柄超発表
  - 8月: 1Q決算 (除外なのでシグナル少)
  - 11月: 中間決算 (2Q) のピーク
- 年あたりエントリー機会: **約2,000件** (全プライム合計)
- 想定年率リターン: 30-50% (¥1500万を効率的に回す前提)

## 8. 既存PEAD戦略との関係

| 戦略 | タイミング | 入口 | 出口 | Sharpe |
|---|---|---|---|---|
| **pre_earnings_drift** | 発表"前" | T-5/T-3 引け翌日寄り | T-1 引け | +2.07 |
| earnings_pead | 発表"後" | T+1 引け | T+5 引け | +2.19 |

→ **同じ銘柄を発表前後で連続Hold可能**:
   - T-5寄り Long (PED) → T-1引け Exit
   - 翌日 (T) 引け後発表 → 翌々日 (T+1) ギャップ判定
   - GU+7%超 → T+1 引け Long (PEAD) → T+5 引け Exit

これにより**「決算前4日 + 発表後5日 = 9日間」の連続エッジ取り**が可能

## 9. 限界・注意点

### 統計的限界
- 1Q決算と業績修正は除外 (機能しない)
- 配当修正は N=107 で小サンプル

### 実装上の注意
- **決算延期リスク**: 予定日が変更されることあり (要日次再判定)
- **寄付ギャップ**: 5/14 のような決算ピーク前日は事前に窓を開ける可能性 → 寄りで高値づかみ
- **キャパシティ**: ¥100万/銘柄なら問題なし、¥1000万/銘柄は要検証

### 既存戦略との重複
- bank_absorption と銀行株で重複の可能性 → サイズ調整推奨
- earnings_pead と同銘柄なら連続Hold
