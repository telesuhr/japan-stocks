# 運用ルール — pair_portfolio

## 実行スケジュール
- **毎営業日 15:35** (引け後) または **翌日 07:00-08:30**:
  - `python3 signal_check.py` を実行
  - 新規エントリー候補と既存ポジのエグジット対象を確認
- **引け成行は使わず**、エントリーは翌営業日の **寄成** で発注

## エントリー
1. `signal_check.py` の `🔔 【新規エントリー候補】` リストに表示されたペアを対象
2. 既存ポジがあるペアは **追加エントリーしない** (重複禁止)
3. 翌営業日 09:00 前に両銘柄を **寄成** で発注
   - Spread Short (Z > +entry_z): **p1 Short / p2 Long** (サイズ比 1:|β|)
   - Spread Long (Z < -entry_z): **p1 Long  / p2 Short** (サイズ比 1:|β|)
4. 発注後、`positions.json` に記録:
   ```json
   {
     "7203.T-7267.T": {
       "entry_date": "2026-04-23",
       "entry_z": -2.49,
       "beta": 0.515,
       "direction": "LONG_SPREAD",
       "notional_ratio": "1:0.515",
       "p1_side": "LONG",
       "p2_side": "SHORT"
     }
   }
   ```

## エグジット
1. `signal_check.py` の `🔻 【エグジット対象】` リストに表示されたポジを対象
2. 翌営業日 09:00 に **両銘柄同時** に **寄成** で反対売買 (**分割 NG**)
3. `reason` 別対応:
   - **MR** (`|Z|<exit_z`): 計画通り利確
   - **STOP** (`|Z|>4.0`): 損切確定。原因ペア (M&A報道・決算サプライズ等) を記録
   - **TIME** (`MaxHold 経過`): 時間切れ強制クローズ
4. `positions.json` から該当エントリを削除
5. `trade_log.csv` に結果を追記

## ポジションサイジング
- 各ペアに **総資金の 1/18** を割当
- 例: 総資金 ¥3,000 万 → 各ペア ¥167 万
  - p1 に ¥167 万 × 1/(1+|β|)
  - p2 に ¥167 万 × |β|/(1+|β|)
- レバレッジは最大 3 倍まで (MDD 262 bps × 3 = 786 bps ≈ 8% の想定)

## 例外・停止条件
- **単月 -500 bps 超 drawdown**: 以下を実施
  1. 全ポジを翌営業日寄成でクローズ
  2. 1 ヶ月運用停止し原因分析
  3. ペア構成の見直し (H2 が悪化した 3 ペア以上を除外)
- **1 ペアが 3 連敗で累計 -300 bps 超**: そのペアのみ停止 (他ペアは継続)
- **市場全体のボラ異常 (VIX ≥ 40)**: 新規エントリー見送り (既存ポジは通常通り管理)
- **個別銘柄に重大イベント** (M&A発表・上場廃止・TOB等): 該当ペアを即座に手動クローズ

## 禁止事項
- ペアに含まれる **片方のみ持ち越し禁止** (ヘッジが消える)
- **分足での執行禁止** (コスト 16 bps に倍増しエッジ消滅)
- **entry_z/exit_z の恣意的変更禁止** (オーバーフィット防止)
- **βの手動上書き禁止** (毎日 trailing 60日 OLS で自動再計算)

## 月次レビュー (毎月末)
1. 該当月の全トレードを `trade_log.csv` で集計
2. 実際の mean/Sharpe/t-stat がバックテスト範囲 (Sharpe 0.8-1.8 / mean 20-100 bps) 内か確認
3. 逸脱していれば原因調査 (個別ペアの崩壊 / 相関上昇 / 市場レジーム変化)
4. `FINDINGS.md` に月次差分を追記

## 四半期レビュー
1. 18 ペアそれぞれの過去 1 年 Sharpe を再計算
2. **Sharpe < -0.5 となったペアは一時停止**
3. 必要に応じて新規ペア候補を追加検討 (`analyses/` 配下で事前検証)
