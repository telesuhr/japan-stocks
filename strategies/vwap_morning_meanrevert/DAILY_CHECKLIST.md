# 毎日のオペレーション手順 — vwap_morning_meanrevert

## 🕘 9:00 (寄付)

- [ ] 本日 FOMC/日銀/SQ/対象3銘柄の決算発表日でないか確認 → **該当時は本日スキップ**
- [ ] 前日 .SOX ±5% 超の急変がないか確認 → **超過時はスキップ**

## 🕙 10:00 — シグナル監視開始

`signal_check.py` を起動、または手動で以下を監視:

3銘柄それぞれで:
1. day-VWAP を 9:00 累積で計算
2. 現在価格との乖離 (bps) を 1分毎に更新
3. **\|乖離\| ≥ 275bps** になった瞬間アラート

```
乖離 = (現在値 / VWAP - 1) × 10000
```

## 🕙 10:00 - 🕦 11:30 — トリガ発生時

### エントリー手順
1. トリガ発生銘柄を特定 (TEL/ディスコ/レーザー のどれか、複数可)
2. 乖離の符号を確認:
   - **+275bps超 (VWAP上に乖離)** → **Short** 成行
   - **-275bps未満 (VWAP下に乖離)** → **Long** 成行
3. サイズ: 1銘柄 ¥300-500万
4. `trade_log.csv` に記録 (date / sym / entry_time / entry_price / vwap_at_entry / dev_bps / direction)
5. その銘柄はその日これ以上発動しない

### 発動見送り判断
- [ ] 当日 9:00-10:00 レンジが過去20日中央値の **2.5倍以上** → 見送り
- [ ] 月間取引上限 10回に到達 → 見送り

## 🕛 11:30 以降 — 監視終了

11:30 までにトリガしなかった銘柄は**本日の発動なし**。ノーポジで維持。

## 🕒 15:22-15:25 — 引け決済

- [ ] 建玉銘柄に **引成** 決済注文を入れる
- [ ] 15:25 約定を確認
- [ ] `trade_log.csv` に exit_time / exit_price / gross_bps / net_bps を追記

### Stop 発動時 (エントリー後 ±400bps 到達)
- 強制決済 (成行)
- `notes` 欄に「Stop -400bps」と記録

## 🌙 取引終了後

- [ ] 本日の P&L 集計
- [ ] 週末時点で週次集計 → 想定Sharpe+6.11 との乖離確認
- [ ] 月末時点で月次集計 → 月間取引数・Sharpe を記録

## 🚨 異常時対応

| 状況 | 対応 |
|---|---|
| シグナル検知スクリプト停止 | 手動でVWAP計算し監視継続 |
| 寄付が遅延 (特別気配継続) | その銘柄は当日スキップ |
| 信用規制 or 売買停止 | その銘柄は当日スキップ |
| 相場急変 (サーキットブレーカー) | **本日全建玉を速やかに決済** |
