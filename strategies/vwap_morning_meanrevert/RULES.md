## 2. VWAP 定義

**9:00 開場からの累積 session-VWAP** (前日分は含まない)

```
VWAP_t = Σ(close_i × volume_i) / Σ(volume_i)     for i in [9:00, t]
```

実装注意:
- 前場 (9:00-11:30) と 後場 (12:30-15:30) を**連続して累積** (昼休みで reset しない)
- 出来高ゼロバーがあれば skip (VWAP更新せず)
- broker提供のVWAPが異なる場合は自前計算優先

## 3. シグナル監視ロジック

### 監視窓
- **10:00 〜 11:30** の 1分バー (前場のみ)
- 各銘柄独立に判定

### 乖離計算
```
乖離_bps = (close / VWAP - 1) × 10000
```

### トリガ条件
- 10:00-11:30 の間で **初めて** \|乖離\| ≥ **275bps** を観測した瞬間
- 一度トリガすれば、その銘柄はその日それ以降ノーアクション

## 4. エントリー

### 方向
- 乖離 > +275bps → **Short** (VWAPより上に乖離 = overbought とみなし fade)
- 乖離 < -275bps → **Long** (VWAPより下に乖離 = oversold とみなし fade)

### 注文
- 検知バーの次バー (9:00-11:30 内) で **成行**
- サイズ: 1銘柄あたり ¥300-500万 (固定)

### 同時発動時
- 3銘柄同時に条件満たせば **全て建てる** (相関ヘッジの観点で問題なし)
- ただし建玉上限 ¥1,500万を超えないよう発動優先順位:
  1. ディスコ (最強)
  2. TEL
  3. レーザー

## 5. エグジット

### 時刻
- **15:25 引成** (一律)

### 早期決済の禁止
- 11:25 前場引けでの決済は **Sharpe -2 以下** → 絶対禁止
- 損切りは後述の Stop のみ

### Stop (オプショナル, 実装推奨)
- エントリー価格 ± **400bps** で強制決済
- バックテスト上は Stop なしだが、テールリスク管理のため設置

## 7. 発動見送り条件 (実運用時の安全装置)

以下のいずれかに該当する日は**発動見送り**:

- [ ] FOMC・日銀会合当日 (終了後17時以降は解禁)
- [ ] SQ日 (毎月第2金曜, 特別清算指数)
- [ ] 決算発表日当日 (該当銘柄のみ)
- [ ] 当日 9:00-10:00 レンジが過去20日中央値の **2.5倍以上** (超高ボラ)
- [ ] 前日 .SOX が ±5% 以上 (米国パニック相場)

## 8. 発注テクニック

### エントリー
- 成行で問題ないが、**トリガバー+1-2分以内に発注**
- 板が薄い場合は 3tick 以内の指値でも可

### エグジット
- 15:25 引成 = 3分前の 15:22 で `引成` 注文入れる
- broker によっては 15:24 以降の引成入力 NG の場合あり、事前確認

## 9. 破綻条件 (戦略停止判定)

以下のいずれかで **戦略を一時停止**し再検証:

- 3ヶ月連続で月次 Sharpe < 0
- 最大DD が -800bps を超える
- 月間取引数が平均の 2倍 (≥10回/月) を 2ヶ月連続
- 銘柄別で1銘柄が突出してマイナス (6ヶ月で Sharpe < -1)
