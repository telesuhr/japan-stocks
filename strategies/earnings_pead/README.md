# PEAD Long 戦略 (Post-Earnings Announcement Drift)

**型**: スイング Long (5営業日保有)
**バスケット**: プライム全銘柄 (除外セクター: 銀行・食料品・金属製品・サービス業)
**昇格元**: [`analyses/20260512_earnings_pead_validation/`](../../analyses/20260512_earnings_pead_validation/)

---

## 1. 戦略の核

### コンセプト
**Post-Earnings Announcement Drift (PEAD)** = 決算サプライズ後の継続上昇現象。
良い決算で大きく窓を開けて上昇した銘柄は、その後5日間も上昇傾向を持続する。

### 経済的根拠
- **情報の遅延織込み**: 機関投資家の分析・レポート発行に時間がかかる
- **アナリスト上方修正の遅れ**: 1-2週間遅れて出る
- **個人投資家の追随買い**: SNS拡散・ニュース読みで遅れて入る
- **インデックス・ファンド調整**: 月末リバランスで組入比率変更
- 米国でも長年実証されている市場アノマリー

### シグナル
**Day N 引け直前 (15:25)** に以下を全て満たした銘柄:
- 前営業日 (Day N-1) 15:00 以降に決算発表 (AC = After Close)
- **Day N の寄付ギャップ ≥ +7%** (= adj_open / prev_close - 1)
- Day N の売買代金 ≥ **5億円**
- 除外セクター (銀行業/食料品/金属製品/サービス業) に該当しない

### エントリー / 出口
| 項目 | 内容 |
|------|------|
| エントリー | Day N **15:30 引成 Long** |
| 出口 | Day N+5 (5営業日後) 15:30 引成 |
| **損切 (SL)** | **-5%** (intraday low が約定価格×0.95 以下) |
| 利食 (TP) | **なし** (上方向は無制限ホールド) |
| ポジションサイズ | ¥1,000,000 / 銘柄 |
| 同時保有数上限 | **10銘柄** |

## 6. 運用ルール

### 日次フロー
1. **15:25** Day N 引け直前
   `python3 signal_check.py` 実行
2. シグナル判定:
   - GO → Day N 15:30 引成 Long で発注 (ギャップ降順で上限10銘柄)
   - SKIP → ノートレード
3. **同時に**: 5営業日前(Day N-5)に建てたポジションを 15:30 引成 Sell

### 損切オペレーション
- 各保有銘柄の **約定価格 × 0.95** を SL ラインに設定
- intraday で SL ラインに到達したら**即時成行 Sell** (寄付以外可)
- バックテスト: 30.8% の確率で SL ヒット、平均 -5% で確定損切

### 重複時のルール
- 既保有銘柄が同条件で再度シグナル発生 → **スキップ** (重複建玉禁止)
- 同時保有 10銘柄に達した場合、新規シグナルはスキップ
- 古い順に決済されて枠が空けば次のシグナルから採択

### 月次メンテナンス
- 毎月初: 直近1ヶ月の実績をログ集計
- net mean > 0、勝率 > 45% を維持しているか確認
- 異常損失 (-10%以上のトレード) の原因分析

### 半年次レビュー
- 24ヶ月バックテスト再実行
- 除外セクター見直し (新たに Sharpe<0 のセクターがあれば追加)
- doc_type 除外見直し (FY IFRSなど)

## 7. 想定資金枠

- 通常運用: ¥10,000,000 (10銘柄 × ¥100万)
- ピーク期 (5月・11月本決算): 同時シグナル多発、最大10銘柄稼働
- 閑散期 (6-7月、9-10月): 月数件のみ
- 年あたりエントリー機会: **約500件** (全プライム)
- 推奨運用期間: 1年以上（季節性を考慮）

## 8. 限界・注意点

### 統計的限界
- 2025年に一時弱化 (Sharpe +0.37)、原因は不明
- 月次・四半期で発動頻度が大きく変動

### 実装上の注意
- **約定リスク**: GU+7%超の銘柄は寄付直後にストップ高近くで張り付くことあり
- **大型株 (30億+) は若干劣化**: Sharpe 2.19→1.55程度。流動性 vs エッジのトレードオフ
- **本決算IFRS発表 (-3.38)** は実装上**除外推奨** (将来 v2 で対応)

### キャパシティ
- 1ポジ¥100万なら問題なし
- ¥1000万/銘柄に拡大する場合はマーケットインパクト要検証
