## 🌅 Day N 15:25 — 引け直前 シグナル判定

### 前準備
- [ ] DB同期完了 (`stocks_daily` の最新日が当日になっているか)
- [ ] `fin_summary` の前営業日(T)分が取り込まれているか
- [ ] 現在の保有銘柄数を確認 (上限 10銘柄)

### 実行
```bash
cd ~/claude-code/japan-stocks
python3 strategies/earnings_pead/signal_check.py
```

### 判定結果ごとの行動

#### ✅ GO (シグナル発生・採択 ≥ 1銘柄)
- [ ] 表示された採択銘柄を確認
- [ ] 各銘柄の以下をチェック:
  - [ ] 制限値幅張り付き (ストップ高) でないか
  - [ ] 売買代金が直近5分でも継続して立っているか
  - [ ] 大型ニュース (買収・MBO等) で異常値ではないか
  - [ ] Day N+1, N+2 にFOMC/日銀会合がないか
- [ ] **15:30 引成 Buy** で発注 (各 ¥100万)
- [ ] 約定確認

#### ❌ SKIP
- [ ] ノートレード確認
- [ ] signals_log.csv に記録されていることを確認

## 📊 保有中 (Day N+1 〜 Day N+5)

### 毎日 09:00 - 15:30
- [ ] 各保有銘柄の現値を監視
- [ ] **現値 ≤ SL価格** → 即時成行 Sell
  - 寄付ですでに SL以下 → 寄成 Sell (gap down損切)
  - 場中 SLヒット → 成行 Sell
- [ ] SL約定後は trade_log.csv に exit_reason=SL / 約定価格 / 日付を記録

### 毎日 15:35 引け後
- [ ] 各保有銘柄の終値ベース評価損益を記録
- [ ] 異常値 (急落・急騰20%超) があれば原因調査
- [ ] 上場廃止・MBO・TOB発表があれば翌日成行 Exit
