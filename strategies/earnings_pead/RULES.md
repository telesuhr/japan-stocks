# 意思決定フローチャート — PEAD Long 戦略

---

## 🟢 Day N 15:25 — 当日エントリー判定

```
START (Day N 15:25 — 引け直前)
  │
  ▼
① signal_check.py を実行
   - 前営業日 (T) 15:00 以降の決算発表を抽出
   - 当日 (T+1=N) の寄付ギャップを計算
  │
  ▼
② シグナル発生銘柄 ≥ 1 か?
  │
  ├── No  → SKIP (skipped_reason=no_signal)
  │
  └── Yes ▼

③ 戦略一時停止フラグが立っているか?
   (直近6ヶ月のSharpe < 0.5)
  │
  ├── Yes → SKIP
  │
  └── No  ▼

④ 現在の保有銘柄数 < 10 か?
  │
  ├── No  → SKIP (枠満杯)
  │
  └── Yes ▼

⑤ シグナル銘柄に既保有銘柄が含まれるか?
  │
  ├── Yes → 該当銘柄を除外
  │
  └── No  ▼

⑥ Day N+1 / N+2 に FOMC / 日銀会合 が控えているか?
  │
  ├── Yes → サイズ半減 (¥50万) または SKIP
  │
  └── No  ▼

⑦ シグナル銘柄に S&P500/TOPIX のSTOP HIGH (制限値幅張り付き) が含まれるか?
  │
  ├── Yes → 該当銘柄除外 (約定リスク)
  │
  └── No  ▼

⑧ ギャップ降順で上位N銘柄を採択
   (N = min(10 - 現在保有数, シグナル銘柄数))
  │
  ▼
GO (採択銘柄を Day N 15:30 引成 Long で発注)
```

```
  │
  ▼

   SL_price = 約定価格 × 0.95
  │
  ▼
③ trade_log.csv に約定価格・SL価格を記録
```

## 🛑 保有中 (Day N+1 〜 Day N+5) — SL監視

```
START (毎営業日 09:00 〜 15:30)
  │
  ▼
① 各保有銘柄の現値を監視
  │
  ▼
② 現値 ≤ SL_price?
  │
  ├── Yes → 即時成行 Sell (exit_reason=SL)
  │         trade_log に SL約定価格・日付を記録
  │
  └── No  ▼

③ Day N+5 (5営業日後) 15:30 に達したか?
  │
  ├── Yes ▼
  │   ④ 引成 Sell (exit_reason=TIME)
  │      trade_log に決済価格を記録
  │
  └── No  → 翌日も継続監視
```

**注**: TP (利食) は設定しない。バックテストで TP=なし が最良 (Sharpe +2.19) と確認済み。

## ⚠️ 戦略停止条件

以下のいずれかで戦略を一時停止:

1. **直近6ヶ月のSharpe < 0.5**
2. **5トレード連続で-5%のSLヒット**（市場環境変化のシグナル）
3. **日銀の大幅金融政策変更** (緊急利上げ等)
4. **暴落相場** (TOPIX 月次 -10% 以上の急落) — 1ヶ月待機

停止解除条件:
- ホールドアウト24ヶ月で Sharpe > 1.5 を確認
- 戦略仮説の再検証完了

## 📅 例外処理

### ケース1: 寄付がストップ高 (約定不可)
- そもそも引け値が制限値幅張り付き → 当日中の他の時間に売買代金が立てば対象
- 場中ストップ高解除されて売買代金 5億+ なら、その時の値で gap_pct を再計算
- 15:25 の判定時点で売買代金が 5億未満 → スキップ

### ケース2: 翌営業日に決算発表が連続
- 同じ銘柄が複数回発表 (例: 業績修正 → 翌日 本決算) → 後発の発表を優先
- ただし既保有なら追加買いはしない

### ケース3: 配当落ち日が保有期間に含まれる
- 配当 / 株主優待落ち日も通常通り保有継続
- 配当・優待相当額は PnL に含めない (シンプル化)

### ケース4: 上場廃止・MBO・TOB
- 発表があれば即時成行 Sell
- 通常の SL/TIME ルールを上書き
