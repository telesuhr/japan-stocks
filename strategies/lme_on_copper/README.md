# 戦略: LME銅 東京時間シグナル → コア5 オーバーナイト

**バージョン**: v1.0 (2026-04-21)  
**ステータス**: 採用  
**根拠分析**: [`analyses/20260421_lme_copper_overnight/`](../../analyses/20260421_lme_copper_overnight/)

---

## 一文で

> **LME銅が東京時間中(JST 9:00〜15:25)に +1.0% 以上上昇した日の大引け(15:30)にコア5銘柄をLong、翌朝9:00寄付で全決済。**

---

## 戦略構造

```
シグナル確認  15:25  LME銅の東京時間変化率を計算
               └─ ≥ +1.0% → 本日エントリー
               └─ < +1.0% → 当日スキップ

エントリー    15:30  コア5銘柄 引成Long (各¥1,000万)

オーバーナイト保有  (~17.5時間)

決済          翌9:00  コア5銘柄 寄成売り (全量)
```

**シグナルと取引の関係**:
- シグナルは**当日のLME東京時間中の変化**を使う（前日・累積ではない）
- シグナル判明は15:25。エントリーは同日15:30。
- 保有は翌朝まで。**純粋なオーバーナイト戦略**。

## LME銅「東京時間変化率」の定義

```
東京オープン価格:
  夏時間 (BST, 3月末〜10月末): JST 09:00 の LME銅価格
  冬時間 (GMT, 10月末〜3月末): JST 10:00 の LME銅価格

東京クローズ価格:
  JST 15:25 直前の LME銅価格

変化率 = (15:25価格 / オープン価格 - 1) × 100
```

## 日次オペレーション

### 当日スケジュール

| 時刻 | アクション |
|------|---------|
| JST 09:00 | LME銅の東京オープン価格を記録 |
| JST 15:20 | LME銅の現在値を確認・暫定変化率を計算 |
| JST 15:25 | **変化率を最終確定** |
| JST 15:25-15:29 | +1.0%以上なら5銘柄の引成注文を発注 |
| JST 15:30 | 大引け約定・約定価格を記録 |

### 翌日スケジュール

| 時刻 | アクション |
|------|---------|
| JST 08:00 | 日経先物の夜間水準・重大ニュースを確認 |
| JST 08:50-08:59 | 5銘柄の寄成売り注文を発注 |
| JST 09:00 | 寄付約定・P&L確定・trade_log.csv に記入 |

### 発注種別

| タイミング | 注文種別 |
|---------|---------|
| 当日 15:30 エントリー | 引成 (CLO / At-the-Close) |
| 翌日 09:00 決済 | 寄成 (OPG / At-the-Opening) |

## エントリー除外条件

| 条件 | 理由 |
|------|------|
| **当日が木曜日** | 木曜LMEアップ日のSharpe = -1.31（曜日アノマリー） |
| 対象銘柄の当日・翌日に決算発表あり | オーバーナイト中の決算サプライズリスク |
| 当日の日経が-2%超の急落 | パニック相場では翌朝さらに売られるリスク |
| 米FOMC・雇用統計の前日 | 夜間重要指標によるテールリスク → サイズ半減推奨 |

| 項目 | 設定 |
|------|------|
| 1銘柄あたり投下額 | ¥1,000万 |
| バスケット合計 | ¥5,000万 |

ONテールリスク(1日最大想定損失):
- 通常変動: ±0.5〜1.5% (±¥25〜75万)
- 夜間サプライズ: -3〜5% (-¥150〜250万)
- ブラックスワン: -10%超 (数年に1回級)

## topix_overnight との関係

両戦略は同じバスケット・同じエントリー・決済タイミングだが、**シグナルが独立している**。

| 項目 | lme_on_copper | topix_overnight |
|------|--------------|----------------|
| シグナル | 当日LME銅東京時間変化率 | 前日終値→当日9:00 TOPIXギャップ |
| 判定時刻 | 15:25 | 09:05 |
| 月間発動回数 | 1〜2回 | 4〜5回 |
| Sharpe | **+11.50** | +4.79 |

**同日発動時**: lme_on_copper を優先（Sharpeが高い）。重複してベットしない。
