### 一文で
> **Day N の東京時間中 (JST 9:00 → 15:25) に LME銅が +1.0% 以上上昇したら、15:30 大引けで コア5銘柄 を等金額Long エントリー → Day N+1 9:00 寄付で全決済。**

### ロジック (なぜ効くのか)
- LME銅の東京時間セッションは、その後ロンドン本番セッション (JST 16:00-翌1:00頃) に引き継がれる
- 東京時間中の強い銅モメンタムは**ロンドン時間中も追随する傾向**が統計的に有意
- 日本株コア5 (景気敏感) は翌朝寄付でこの追随ムーブを**ギャップアップで反映**する
- エントリー (15:30) → 決済 (翌9:00) の **17.5時間** でそのギャップアップを捕捉

## 1. エントリーシグナル

### 条件 (AND)

```
IF  LME銅 (CMCU3) の東京時間変化率 ≥ +1.0%
AND Day N が 木曜日ではない
AND 対象銘柄が決算発表日ではない (Day N または Day N+1)
THEN Day N 15:30 引成でエントリー
```

### LME銅「東京時間変化率」の定義

東京時間中のLME銅価格変化を使う (ロンドン終値ではない):

```
東京時間オープン価格:
  夏時間 (BST, 3月末〜10月末): JST 09:00 のLME銅 open
  冬時間 (GMT, 10月末〜3月末): JST 10:00 のLME銅 open

東京時間クローズ価格:
  JST 15:25 直前 (日本株の引け直前) の LME銅 close

変化率 = (東京クローズ / 東京オープン - 1) × 100
```

### 確認タイミング

| 時刻 | アクション |
|---|---|
| JST 15:20 | LME銅の東京時間オープン/直近価格を確認 |
| JST 15:25 | 変化率を確定 |
| JST 15:25 〜 15:29 | シグナル発動なら日本株発注 |
| JST 15:30 | 引成で約定 |

### 閾値の根拠

バックテスト結果 (threshold 感応度):

| 閾値 | N (年間) | 代表Sharpe | コメント |
|---|:-:|:-:|---|
| +0.5% | 104 | 2-4 | 弱シグナル |
| +0.8% | 50 | 3-5 | 中 |
| **+1.0%** | **19** | **6-11** | **最適域 (採用)** |
| +1.5% | 14 | 3-5 | N少で統計脆弱 |

### 発注種別の指定

| タイミング | 注文種別 | 備考 |
|---|---|---|
| Day N 15:30 エントリー | **引成 (成行引け / CLO)** | 引けオークション参加 |
| Day N+1 09:00 決済 | **寄成 (成行寄付 / OPG)** | 寄りオークション参加 |

**理由**: 引け/寄りはザラ場中より流動性集中ポイント。オークション約定で実効平均価格に近く約定、スリッページ最小。

### 発注テンプレート

**エントリー時 (Day N 15:25頃)**:
```

銘柄: 5711, 6501, 7011, 5016, 4502 (各々)
数量: 事前計算した株数
価格: 成行
執行条件: 引成 (CLO / At-the-Close)
期間: 当日限り
```

**決済時 (Day N+1 08:50頃)**:
```
注文区分: 現物売り (決済)
銘柄: 5711, 6501, 7011, 5016, 4502 (各々)
数量: 保有株数 全量
価格: 成行
執行条件: 寄成 (OPG / At-the-Opening)
期間: 当日限り
```

## 4. 例外処理・フィルター

### 🚫 Day N エントリー見送り条件

| 条件 | 理由 | 対応 |
|---|---|---|
| **Day N が木曜日** | 曜日アノマリー Sharpe-1.31 | 全銘柄スキップ |
| 該当銘柄の決算が **Day N または Day N+1** | ON中に決算で予想外の動き | その銘柄のみ除外 |
| Day N の日経終値が前日比 **-3%以上の急落** | パニック相場で翌日さらに売られリスク | スキップ推奨 |
| 対象銘柄がストップ安・売買停止 | 執行不可 | その銘柄のみ除外 |
| Day N が**配当落ち前日** | Day N+1寄付が配当分下がる | その銘柄除外 (配当調整後で別管理) |
| 米雇用統計前日・FOMC前日 | 夜間の重要指標でリスク急増 | サイズ半減 推奨 |

### 🚨 Day N+1 決済前の特殊対応

| 状況 | 対応 |
|---|---|
| 日経先物夜間 -3% 超で寄付 | 予定通り寄成売り (損失確定) |
| 個別銘柄に決算・不祥事 → ストップ安気配 | 値幅制限まで待機、決済翌日に持ち越し (ルール外) |
| 米国市場が -5% 超の大暴落 | 寄付前に日経ミニ先物でヘッジ検討 (裁量) |

### 配当落ち日の計算

対象銘柄が配当落ちでエントリーした場合:
```
調整後リターン = 生リターン + (配当額 / エントリー価格)
```

主要配当月 (CORE5): 3月, 9月 — 特にこの月の月末には注意

## 5. リスク管理

### ポジション上限

| 項目 | 上限 |
|---|:-:|
| 1銘柄あたり投下額 | ¥1,000万 |
| 1日あたり総建玉 | ¥5,000万 |
| 月間損失ストップ | -¥200万 (4%) で月内新規停止 |
| 年間損失ストップ | -¥500万 (10%) で年内撤退 |

### ONリスクの許容範囲

ONで取る最大想定損失 (1日):
- 通常変動: ±1-2% (±¥50-100万)
- テールイベント: -5% ≒ -¥250万 (超稀)
- 暴落日 (2020年3月コロナ級): -10% ≒ -¥500万 (10年に1回級)

→ **単日損失の95%ile は -2% 以内**。リスクは限定的だが、ONゆえにゼロではない。

### 損切ルール

**ON中は原則ノータッチ** (損切を入れると期待値低下):
- バックテストでは N=19 のうち負けトレードの平均DDは約 -80bps
- 個別損切を入れると連続勝ち局面で利益も限定される

**手動介入する場合の目安**:
- 夜間に日経先物が -3% 超下落 → 翌朝寄成でロスカット (予定通り)
- 個別銘柄が夜間PTSで -5% 超売られる → 該当銘柄のみ翌朝ロスカット検討

## 8. トラブルシューティング

### Q: LME銅の JST 9:00 オープン価格がDBに入っていない
A: 直近の JST 9:00 以降の最初の価格を使用。通常 9:00-9:05 以内にデータが入る。

### Q: JST 15:25 時点で +1.0% 超えていたが、15:30 引けで+0.9%になった
A: **15:25 時点の確定値を採用**。エントリーした以上は翌朝まで保有。

### Q: 引成で1銘柄だけ約定しなかった
A: ザラ場最終値付近で板を見て成行で買いなおし (15:30直後 OK)、または当日スキップ。記録には "partial_fill" と注記。

### Q: Day N+1 の寄付が特別気配で遅れた
A: 寄付成立を待つのみ。9:30 までに成立すれば OK。遅延が長い場合は成立後すぐ決済。

### Q: Day N+1 に個別銘柄が S安気配 (寄らず)
A: 当日寄らなければ持ち越し (ルール外の特殊処理)。翌々日に寄成で決済。ログに "escalated_close" と記録。

### Q: 夜間に米国市場が大幅安 (-5%超) で、寄付前に決済したい
A: 現物株は寄付前決済不可。日経ミニ先物で一時ヘッジ → 9:00寄付で現物売り & 先物買戻し。ただしこれは裁量枠、年間2回以内に限定。

### Q: シグナル発動だが「何となく相場がおかしい」
A: スキップ可。`skipped_reason=discretionary` と記録。年3回まで。
