# ORB Fade 条件最適化 (❌ 不採用)

## 目的
[`20260422_semi_intraday_battery/`](../20260422_semi_intraday_battery/) で ③b ORB Fade が
Pooled Sharpe+1.01 / t=+2.15 (N=1130) を示したため、条件層別で Sharpe≥+2.0 に
引き上げられるか検証。

## 手法
半導体5銘柄 (6857/6920/6146/6503/8035) × 2024-11〜2026-04 の 1分足データで、
9:00-9:30 Opening Range を定義し、9:30 以降の H/L ブレイクに対して **逆方向にエントリー**。

### グリッド
- `or_min`: OR幅下限 0 / 50 / 100 / 150 / 200 bps (小ブレイクを除外)
- `cutoff`: ブレイク受付時刻 11 / 13 / 15時まで
- `hold`: 保有時間 30min / 90min / 引け (close)
- `TP / Stop`: TP = 反対側 OR、Stop = エントリー ± stop_mult × OR幅 (1.0)
- コスト: 4bps

**battery (Sharpe+1.01) との実装差分**:
battery は forward P&L に対し `-ret - 2*COST_BPS` でサイン反転 → これは実質的に
「forward の利益/損失を逆転させた仮想P&L」であり、Fade 実取引時の TP/Stop 構造と一致しない。
本最適化では**明示的な TP (反対OR) / Stop (OR幅の倍数)** を実装。

## 結果 (グリッド全46パターン)

| 指標 | Best | Worst | Median |
|---|:-:|:-:|:-:|
| Sharpe | **-0.91** | -2.7 | -1.6 |
| t-stat | -1.40 | -5.1 | -2.9 |

**全46パターンで Sharpe < 0**。最良でも Sharpe-0.91。TP/Stop を外して単純引け決済でも
Pooled Sharpe ≈ -0.13。Sharpe≥+2 は到達不能。

### 銘柄別 (TP=反対OR / Stop=OR幅 / 15:30close / OR幅フィルタなし)

| 銘柄 | Sharpe | 判定 |
|---|:-:|:-:|
| 8035 東エレク | +1.21 | 唯一プラスだが単独では基準未達 |
| 6503 三菱電機 | ≈ 0 | ノイズ |
| 6857 / 6920 / 6146 | 負 | 逆張り機能せず |

## 結論

### battery の Sharpe+1.01 は実装アーティファクト
battery では ③ 順張りの強い負エッジ (Sharpe-1.89, t=-4.01) を単純にサイン反転しただけで、
これは「forward が Stop にヒットした時、fade は同じ値幅の利益として計上される」という
**非現実的な P&L 構造** を内包していた。実取引で成立する TP (反対OR) / Stop (OR幅倍) を
明示実装すると、エッジは消失する。

### ORB 順張りの負 t=-4.01 をどう解釈するか
「騙しブレイクは多い」という事実は残るが、それを**実務的に収益化できる逆張り仕様は
本グリッドでは見つからなかった**。おそらく:
1. 真に反転するブレイクより、Stop にヒットしてしまう継続ブレイクが多い
2. 反転する場合も TP (反対OR) 到達前に戻ることが多い

## 不採用判断

- ❌ Sharpe≥+2.0 (採用基準) 到達不能
- ❌ H1/H2 OoS 両期間で負
- ❌ 銘柄別でも 8035 単独でようやく +1.21 (単独では N 不足・基準未達)

**ORB Fade 戦略は本プロジェクトでは採用見送り**。今後は別のエッジ源を探索する。
