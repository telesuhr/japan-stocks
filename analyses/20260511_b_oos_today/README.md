## 何が起きていたか

### 大口買いの正体は「買い向かい (失敗)」

通常、B戦略が機能するシナリオは:

```
機関が「上昇トレンド予測」で大口買い投入
  ↓
板に買い圧力が継続
  ↓
価格がジワジワ上昇 (60分間ドリフト+30bps)
```

しかし 5/11 は逆だった:

```
寄付からマイナス圧力で板が薄い
  ↓
誰かが「ここで反発する」と読んで大口買い
  ↓
売り圧力の方が強く、価格はさらに落ちる
  ↓
買い向かいの失敗、損失拡大
```

つまり **「大口買い = ファンダメンタル買い」** という仮説は、**下落トレンド日には成立しない**。

### 出来高基準の問題

学習期間のSBG 99%ile = **8,200株** を閾値に使ったが、5/7-5/11 は出来高が異常に膨らんだ局面 (5/7 出来高119M = 通常の1.6倍)。

→ この期間の本当の99%ile は 12,000株 以上だった可能性。  
8,200株は「**ノイズの平均レベル**」になっていて、機関フローを抽出できていない。

## なぜこれが起きるのか (B戦略の構造的な脆さ)

1. **損切りなし**: 60分強制ホールドが致命的。-380bps級の損失が出ても止めない
2. **片方向のみ**: ロングオンリーなので下落相場で逆風固定
3. **シグナル定義の静的閾値**: 99%ile が固定なので異常レジームでは大口判定が崩壊
4. **トレンドフィルタなし**: 「今日の地合いが下げか上げか」を見ない

---

## 今後の必須改善

### Phase 1: レジームフィルタ追加

```
追加条件:
  - 寄付直後30分の累積騰落率が -100bps を下回ったら新規エントリー停止
  - 前日終値 → 寄付ギャップが -200bps 以下ならその日は完全停止
  - TOPIX or 日経先物 の5分足が大きく下落していたら停止
```

### Phase 2: 動的な大口閾値

```
99%ile を「過去5営業日のローリング」で計算
→ 出来高が膨らんだ日は閾値も自動で上がる
```

### Phase 3: 損切り再導入 (深掘り分析で否定したが再考)

```
- 30分時点でMFE (最大利益) がマイナスのまま → 早期撤退
- 累積DD が -50bps を超えた時点で全ポジションクローズ
```

### Phase 4: 上下両建て対応

```
朝の地合いを見て:
  上昇トレンド → 大口買い追随 (ロング)
  下落トレンド → 大口売り追随 (ショート) ← B4 の利用
```

## 教訓 (重要)

1. **4ヶ月のバックテストでは"急落日"が十分カバーされていなかった**: 2026-03 の負け月はあったが、5/11 級の暴落はなし
2. **Sharpe=18 という数字は嘘ではなかったが、それは「通常の市場環境」の話**: 平均的なリターンは確かに高いが、テールリスクが極端
3. **イベント駆動の銘柄 (SBG) はイベント日にこそ大損する**: 学習期に「決算がない静かな期」だっただけ
4. **OOS検証 1日でも価値ある**: バックテストの過信を1日で粉砕した
