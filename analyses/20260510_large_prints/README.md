# B. 大口約定の方向性追随

## 目的・仮説

ティックデータの個別約定レベルで「大口プリント (機関投資家フローの可能性)」を抽出し、その方向に追随できるかを検証。

仮説:
- H1) 大口買い (uptick で大量株数) → 同方向に継続 (機関買いの追随)
- H2) 大口売り → 同方向に継続
- H3) 大口プリントは過剰反応 → 反転する

---

## データ概要

- 期間: 2026-01-05 〜 2026-04-30 (4か月)
- 銘柄: トヨタ・SBG・MUFG・アドバンテスト・ソニーG (流動性主要5銘柄)
- 元データ: JQuants ティック (個別約定)
- **大口閾値**: 各銘柄の出来高99%ile以上の単発約定 (例: トヨタ ≥10,400株、SBG ≥8,200株)
- 抽出した大口プリント: **118,151件** (アップティック59,403 / ダウンティック58,691)
- コスト: 往復4bps

### 方向推定 (Lee-Ready 簡略版)

| 直前ティック比 | 方向 | 解釈 |
|---|---|---|
| price > prev | +1 (uptick) | 買いに叩かれた = 買い主導 |
| price < prev | −1 (downtick) | 売りに叩かれた = 売り主導 |

## 核心的発見: 「買い」と「売り」の完全な非対称性

### 大口買いの方向 → そのまま追随OK

| 経過時間 | mean_raw (bps) | net (bps) | t値 | 解釈 |
|---|---|---|---|---|
| 1分後 | +0.04 | -3.96 | 0.41 | コスト負け |
| 5分後 | +0.69 | -3.31 | 3.79 | エッジあり、コスト負け |
| 15分後 | +2.61 | -1.39 | 9.14 | エッジあり、コスト負け |
| **30分後** | **+6.34** | **+2.34** | **16.89** | ✓ コスト超え |

→ **大口買いプリント直後、価格は30分かけて+6bps上昇する**。コスト4bpsを引いても +2.34bps の純益。

### 大口売りの方向 → 追随したらダメ (反転する)

| 経過時間 | B2 (売り継続) raw | B4 (反転買い) raw | 解釈 |
|---|---|---|---|
| 1分後 | -0.32 | +0.32 | 微弱反転 |
| 5分後 | -0.71 | +0.71 | 反転継続 |
| 15分後 | -1.48 | +1.48 | 反転拡大 |
| 30分後 | -4.01 | +4.01 | **明確に反転 (t=10.85)** |

→ **大口売りプリント直後、価格は30分かけて+4bps「反発」する**。売り追随 (B2) は -8bps コスト後で大負け。

---

## 解釈: なぜ非対称なのか

### 大口買い = ファンダメンタル買い (続く)
- 機関投資家が確信を持って買いに行く動き
- 一発で完了せず、続きの買いが入る
- 価格は徐々に上がる (16.89σ という非常に強いシグナル)

### 大口売り = ストップロス・パニック (戻る)
- 強制決済 / マージンコール / 損切りの一括売り
- 一発出し切ると後続の売りがない
- 押し目買いが入って反転 (10.85σ)

→ 「**大口プリントは買いと売りで意味が違う**」という極めて重要な発見。

---

## 前回 (A) との比較・統合

A (1分バーのティック数スパイク) と B (個別大口プリント) は別々のシグナル:

| シグナル | 下落方向 | 上昇方向 |
|---|---|---|
| **A: ティック数スパイク** | 継続 (+6.55bps net @30min, t=2.88) | 予測力なし |
| **B: 大口プリント** | 反転 (B4: +0.01bps net @30min, t=10.85) | 継続 (+2.34bps net @30min, t=16.89) |

つまり下落方向の振る舞いが**シグナル種別で逆**:
- 多数の小口がパニック → 続落 (A2)
- 単発の大口がドーン → 反発 (B4)

これは A の 「下落時のパニック売り」と B の「大口の処分売り」が**異なる現象**であることを示している。

組み合わせると:
- ティック数スパイク + 下落 + 大口プリントなし → 売り (パニック継続)
- 大口下落プリント + ティック数低 → 買い (処分終了、反転)

## 実装イメージ (最有力: B1)

```
監視: 全ティックを ストリームで読み込み
条件: 単発約定が直近の99%ile出来高以上
       AND 直前ティックより高値 (uptick)
エントリー: 当該約定価格でロング (成行)
エグジット: 30分後の最良値で売却
期待: +2.34bps/トレード, Sharpe=3.14
頻度: 5銘柄合計で月15,000回 (銘柄あたり1日150回程度)
```

シグナル頻度が高すぎる場合は閾値を 99.5%ile / 99.9%ile に上げて絞り込み可能。

---

## 限界・注意点

1. **Lee-Ready 簡略版**: 直前比のみで方向推定。本来は気配値とのbid/ask差で判定すべきだが、JQuantsティックには気配が含まれない
2. **コスト 4bps の妥当性**: 1日150回もエントリーすると現実的にスリッページが拡大する
3. **5銘柄に限定**: 中小型株では結果が異なる可能性
4. **30分保有のリスク**: 場中ニュースで急反転するリスク
5. **N=59,403 の独立性**: 大口プリントは時間的にクラスター化するため、実効N(独立サンプル数)は名目より少ない

---

## 次のアクション候補

1. **A+B 統合シグナル**: 「大口買い + ティック数スパイク同時」で絞ったケース → 強いはず
2. **B1の銘柄別検証**: トヨタとSBGでエッジが同じか
3. **時間帯フィルタ**: 寄付直後 / ランチ前 / 大引前 で挙動が異なるはず
4. **保有時間最適化**: 30分が最適か、もっと長く (1時間、EOD) するとどうか
