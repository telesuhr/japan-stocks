# B-Improved: レジームフィルタ追加版

## 目的

5/11 で B-Optimized が 107戦全敗・-40,700bps 大惨敗。  
原因: 「大口買い = ファンダ買い」仮説がダウントレンド日に成立しない。  
→ 当日の地合いを判定するフィルタで損失を防げるか?

---

## 検証フィルタ

| ID | 内容 |
|---|---|
| F1 | 寄付ギャップ <= -200bps なら当日完全停止 |
| F1strict | 寄付ギャップ <= -100bps なら当日完全停止 (より厳しい) |
| F2 | シグナル時点の寄付からの累積騰落 <= -100bps なら以降停止 |
| F2strict | 寄付からの累積騰落 <= -50bps なら以降停止 |
| F3 | 動的閾値 (過去5日ローリング 99%ile) を出来高基準として採用 |

---

## 主要発見

### 1. **5/11 はギャップフィルタでは止められなかった**

| 項目 | 値 |
|---|---|
| 5/8 引け | 6,131円 |
| 5/11 寄付 | **6,219円** (= **ギャップ +144bps**, プラスギャップ!) |
| 5/11 引け | 5,743円 (-7.65%) |

5/11 は実は寄付高で始まり、その後崩れた日。**寄付時点では「下落地合い」と判断できない**。
→ F1 (gap<-200), F1strict (gap<-100) ともに**通過してしまう**。

### 2. フィルタ単独効果 (全期間 Train+OOS)

| 設定 | N | mean_net | t値 | 勝率 | Sharpe | 合計net |
|---|---|---|---|---|---|---|
| Baseline | 8,337 | +23.06 | 13.06 | 51.1% | 14.99 | +192,280 |
| F1: gap > -200bps | 6,778 | +31.81 | 15.05 | 54.8% | 19.97 | +215,585 |
| F1strict: gap > -100bps | 6,221 | +35.15 | 15.63 | 55.0% | 21.88 | +218,641 |
| F2: cum_ret > -100bps | 6,157 | +35.72 | 15.96 | 54.3% | 22.49 | +219,904 |
| **F2strict: cum_ret > -50bps** | **5,405** | **+39.84** | **16.27** | **55.2%** | **24.74** | +215,335 |
| F3: 動的閾値 | 7,576 | +21.03 | 11.61 | 51.0% | 13.78 | +159,295 |
| F1+F2 | 5,073 | +42.16 | 16.30 | 57.1% | 25.71 | +213,881 |
| **F1strict+F2strict** | **4,295** | **+47.57** | **16.57** | **57.6%** | **28.69** | +204,329 |

→ 一番効くのは **F2strict (寄付から-50bps以下になったら停止)**。  
F1strict+F2strict 統合で **Sharpe 28.69** (Baseline 14.99 から **約2倍**)。

### 3. 5/11 救済効果 (損失の何%削れたか)

| 設定 | 5/11 N | 5/11 mean_net | 5/11 total_net |
|---|---|---|---|
| Baseline | 110 | -384.5bps | **-42,293bps** |
| F1 (gap>-200) | 110 | -384.5bps | -42,293 (**変化なし**) ← ギャップでは止まらない |
| F1+F2 | 61 | -374.5 | -22,844 (-46%) |
| F1strict+F2strict | 52 | -351.2 | **-18,264 (-57%)** |

→ ギャップフィルタ単独では救えない。「**朝の累積騰落フィルタ**」が9:00以降のシグナルを順次停止することで**5/11の損失を半減**できる。

### 4. F3 (動的閾値) は効果薄

過去5日ローリング99%ileを使う方法は、Sharpe 13.78 と Baseline と大差なし。  
理由: 5/7-5/11 は出来高が急増した期間で、動的閾値も自動で上がる → 結局多くの大口買いを拾ってしまった。

---

## エクイティカーブの変化

```
Baseline   : 学習期+5/11 で  +192,280 bps
B-Improved : 学習期+5/11 で  +204,329 bps  (5/11 損失大幅縮減)
```

5/11 単日の損失:
- Baseline: -42,293 bps (4ヶ月の利益の20%を1日で吐き出し)
- Improved: **-18,264 bps** (痛いが Baseline よりはマシ)

---

## 限界・残る課題

### 1. **5/11 で依然ゼロ勝率**
フィルタで件数を 110→52 まで減らせても、その52件は**勝率0%**。  
→ 戦略の根本的な問題: 「9:00 直後の数分間のシグナルがフィルタを通過してしまう」

```
09:00:00 価格6,220 (寄付直後、累積騰落 +0)
  → F2strict 通過 → エントリー
  → 60分後: 6,054 (-260bps 損失)
```

寄付直後の数分は累積騰落が小さく、フィルタが効かない。これがクリティカル。

### 2. **より強力なフィルタ案**

- **TOPIXフィルタ**: 日経 or TOPIX の現在値が前日比 -200bps 以下ならエントリー停止
- **ボラティリティフィルタ**: 当日寄付30分の HL レンジが 5日平均の2倍超なら停止 (パニック相場の検出)
- **ニュース駆動フィルタ**: 決算カレンダー or ニュース API でイベント当日は停止
- **両建て対応**: ダウントレンド検出時はB戦略を停止 + 売り側戦略 (B4) に切替

---

## 結論

| 項目 | 評価 |
|---|---|
| 通常日のパフォーマンス | **改善** (Sharpe 15→29) |
| 5/11 のような暴落日 | **完全救済できず** (-18,264bps の損失残る) |
| 実戦投入判定 | **保留**: より強いフィルタ (TOPIX等の外部指標) が必須 |

良い改善ではあるが、5/11級の暴落を完全に避けるには別軸の判定 (市場全体・セクター指数・ボラ環境) が必要。
