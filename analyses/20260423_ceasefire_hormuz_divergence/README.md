## 6. 分析上の気付き・データ品質

### 主要銘柄の未収録 (intraday_data)

CLAUDE.md 記載の主力非鉄 (5706/5711/5713/5714/5801/5802) は **intraday_data にいずれも未収録**。90銘柄収録で欠けている:

```
未収録: 1662(石油資源開発), 1963(日揮HD), 1965(テクノス),
        5016(JX金属), 5019(出光), 5706(三井金属), 5711(三菱マテリアル),

        6146(ディスコ), 6208(石川島汎用), 6366(千代田化工), 6857(アドバンテスト),
        7011(三菱重工), 7721(東京計器)
```

→ **NAS 側の収録銘柄リスト (symbol_master?) の見直しが必要**。非鉄・プラント・防衛の主力が抜けているのは痛い。

### 時刻データの扱い

全テーブルで `timestamp` / `snapshot_time` / `published_at` は **UTC naive で NAS 格納** → PG TIMESTAMPTZ に UTC として流し込み → `AT TIME ZONE 'Asia/Tokyo'` で JST 変換、で正しく動作することを確認済み。

## 付録: 再現 SQL

### 時間別銅・原油・日経終値

```sql
SELECT EXTRACT(hour FROM (timestamp AT TIME ZONE 'Asia/Tokyo'))::int AS h,
       (ARRAY_AGG(close ORDER BY timestamp DESC))[1] AS close
FROM intraday_data
WHERE symbol = 'CMCU3'
  AND (timestamp AT TIME ZONE 'Asia/Tokyo')::date = DATE '2026-04-23'
GROUP BY h ORDER BY h;
```

### セクター別本日騰落

```sql
SELECT symbol,
       (ARRAY_AGG(close ORDER BY timestamp ASC))[1]  AS open,
       (ARRAY_AGG(close ORDER BY timestamp DESC))[1] AS last,
       100.0 * ((ARRAY_AGG(close ORDER BY timestamp DESC))[1]
                - (ARRAY_AGG(close ORDER BY timestamp ASC))[1])
               / (ARRAY_AGG(close ORDER BY timestamp ASC))[1] AS pct
FROM intraday_data
WHERE symbol LIKE '%.T'
  AND (timestamp AT TIME ZONE 'Asia/Tokyo')::date = DATE '2026-04-23'
GROUP BY symbol
ORDER BY pct;
```

### 重要ヘッドライン抽出

```sql
SELECT (published_at AT TIME ZONE 'Asia/Tokyo')::time(0) AS jst,
       category, headline
FROM refinitiv_news
WHERE (published_at AT TIME ZONE 'Asia/Tokyo')::date = DATE '2026-04-23'
  AND headline ~* '(ceasefire|sanction|hormuz|waiver|extends)'
ORDER BY published_at ASC;
```
