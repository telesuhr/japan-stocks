# B-Optimized 執行レイテンシ感度

## 目的

B-Optimized (SBG単独 / 9:00-10:30 / 60min保有) のバックテストは「ティック検出と同時に約定」する理論前提だが、**現実は遅延がある**。検出 → アルゴ処理 → 発注 → 約定 まで時間が経過し、その間に値が動く。

何ミリ秒の遅延までエッジが残るかを実測。

---

## 方法

- 対象: SBG (99840) ティック 5,437,502件
- シグナル: B-Optimized 条件で抽出した 11,655件 (大口買い × 9:00-10:30)
- 各レイテンシで「シグナル発生時刻 + N ms」以降の最初のティックをエントリー価格とする
- エグジットは「エントリー時刻 + 60分」以降の最初のティック価格
- レイテンシ: 0ms / 100ms / 500ms / 1s / 3s / 10s / 30s / 60s

---

---

## 主要発見

### 1. **B-Optimized はレイテンシに極めて強い**

- 0ms → 1秒: Sharpe -0.10 (ほぼゼロ)
- 0ms → 10秒: Sharpe -0.92 (-4%)
- 0ms → 60秒: Sharpe -4.49 (**-19.6% でも 18.44 残る**)

これはHFT戦略ではなく**「機関フローに乗る中速戦略」**であることを意味する。

### 2. **エッジの正体は60分間の遅いドリフト**

レイテンシ別の損失を見ると:
- 最初の60秒で **7.65 bps消失** (全体の22%)
- それ以降は緩やかな減衰

つまり大口プリント直後の最初の数十秒で「素早い反応の効果」がある程度消費されるが、その後の **長い遅延ドリフト** がエッジの本体。これは時間が経っても残る。

### 3. **実用的な意味**

| 環境 | 想定レイテンシ | 実用度 |
|---|---|---|
| 自宅PC + JQuants Live API + 証券会社APIで自動執行 | 1〜3秒 | **十分実用** (Sharpe 22.7) |
| 自宅PC + 証券会社のWebSocket手動執行 | 5〜10秒 | **十分実用** (Sharpe 22) |
| 大口プリントを画面で見て手動発注 | 30〜60秒 | **まだ実用** (Sharpe 18.4) |
| 業務終了後にDayの記録を見て翌日エントリー | 数時間 | ✗ アウト |

---

## 解釈: なぜこんなにレイテンシ耐性が高いのか

### A) シグナルの「期待寿命」が長い

B戦略は **「機関の発注の途中段階」** を捉えている。機関は10万株 / 100万株単位の建玉を一発では成行できないので、何分・何十分もかけて段階的に注文を出す。最初の大口プリントはその「氷山の一角」。

→ シグナル発生から**数十秒程度では機関の発注フローはまだ続いている**ので、遅れて入っても波には乗れる。

### B) HFT が削り取る部分はそもそも少ない

仮に高頻度業者が同じシグナルを 10ms で捕捉して買いに行っても、彼らが取れるのは最初の1〜2bps程度。残り **+30bps は 1分〜60分かけてゆっくり実現**するため、遅れて入る人にも十分残っている。

### C) データの非対称性

JQuantsティックは**配信遅延**があるので (おそらく数百ms〜数秒)、 そもそも全員が「遅延ありの世界」で見ている。プロのコロケーション勢でない限り、皆条件が同じ。

---

## 但し: 別途考慮すべき現実的コスト

レイテンシ感度は楽観的に出たが、以下は**未考慮**:

### 1. 板の薄さ・スプレッド

- 大口プリントの直後は板が荒れている (買い側の指値が消費されている)
- 100株成行で買うのと 1000株成行で買うので、約定価格が違う
- このコストは「マーケットインパクト」で **+1〜3bps** 推定

### 2. JQuants ティックの遅延

- 我々が使っているのは「**過去のティック履歴ファイル**」(月次CSV.gz)
- リアルタイムで使うには JQuants Live API (リアルタイム配信) が必要
- リアルタイム配信は契約プラン次第で **遅延が変わる** (Premium で 数秒、Standardで 1分以上)
- 1分遅延の場合 → Sharpe ~18 はキープできるが、エッジは約20%減る

### 3. 自分の発注によるマーケットインパクト

- 大口買いプリント (8,200株) と同じ方向に自分が成行で乗ると、板を薄くする
- ロットが小さい (100-500株程度) なら影響は無視できる
- ロットを大きくする (5,000株+) と急激に Sharpe が悪化する

## 修正された実用Sharpe推定

| 想定設定 | レイテンシ | スリッページ | 推定 Sharpe |
|---|---|---|---|
| プロ機関 (FIX直結) | <100ms | 1bps | ~21 |
| クラウド自動執行 | 1秒 | 2bps | ~17 |
| 自宅自動執行 (REST) | 3秒 | 2bps | ~17 |
| 自宅手動発注 | 10秒 | 3bps | ~15 |
| **JQuants Live (1分遅延)** | 60秒 | 2bps | **~14** |

「**自宅で自動執行 → 実用Sharpe 15-17**」というのが現実的な期待値。  
これでも年率Sharpe 1.5-2 程度の戦略になり、十分価値がある。

---

## 限界・注意点

1. **4ヶ月のデータ**: レイテンシ耐性が時系列で安定するかは別途検証
2. **エントリー価格は「次のティック」**: 板の最良値ではない。実際の成行ではスプレッドぶん不利になる
3. **エグジットも次のティック**: 60分後にビッド成行で売る前提。スリッページ未反映
4. **コスト4bps固定**: 実際は片側1.5-3bps × 2 = 3-6bps 幅で振れる
5. **自分のオーダーが市場に影響しない前提**: ロットが大きいと崩れる
