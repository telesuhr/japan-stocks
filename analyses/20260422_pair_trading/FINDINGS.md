# ペアトレード分析 — 最終結論

## データ
- 日次終値 2015-01-05 〜 2026-04-22 (2,761日 / 約11年)
- ソース: MariaDB `refinitiv_news.daily_data` (NAS 100.92.181.92)
- 銘柄: 45 (日経大型株中心、セクター横断)
- コスト: 8 bps/往復 (片側 4bps × 2銘柄) をベースライン

## 検証ステップと結果

### Phase 1: 日次 Z-score MR (素朴版)
- 全期間サンプルで β 推定 → look-ahead バイアスで見かけ 100% WR → バグ発見
- **教訓**: ペア戦略ではβ推定も必ず walk-forward 化が必要

### Phase 2: Walk-Forward (trailing 60日 β)
- **単一ペア**では厳格基準 (Sharpe≥2 & N≥50 & H1/H2 両プラス) を **どのペアも満たさない**
- 最良: 銀行 MUFG/SMFG (Sharpe 1.23, H1 +0.82 / H2 +1.59, N=45)
- 次点: 電機 日立/三菱電, エネルギー INPEX/ENEOS, いずれも H1/H2 で安定性欠く
- → **「単独ペアで儲かる統計的アービトラージ」は日本株では困難**

### Phase 3a: 統合ポートフォリオ (★採用候補★)
**18 ペア Equal-Weight ポートフォリオ**で劇的改善:

| 指標 | 値 |
|---|---|
| **Full Sharpe** | **+1.37** |
| **t-stat** | **+4.55** |
| H1 Sharpe | +1.36 |
| H2 Sharpe | +1.39 |
| 累積 PnL (11年) | +3,400 bps (34%) |
| Max Drawdown | 262 bps |
| **年次プラス率** | **11年中 全年プラス** |
| 平均ペア間 \|ρ\| | 0.008 (ほぼ無相関) |

個別ペアは Sharpe 0.1〜0.6 で凡庸。しかし相関がほぼゼロなので √18 ≈ 4.2 倍に Sharpe が増幅 (教科書的な統計的アービトラージの挙動)。

### Phase 3b: セクター内 L/S バスケット (失敗)
- 横断面 Z-score (lookback 20日) で losers long / winners short
- Sharpe **-0.16** (逆張り) / -0.23〜-0.32 (モメンタム)
- → セクターあたり銘柄数 (3〜9) が少なすぎシグナルがノイズに埋もれる

### Phase 3c: フィルタ付きペア MR (部分的改善)
ベースラインに対し:
- **Regime+Vol フィルタ**で 1 トレードあたり +23% のエッジ向上 (61→75 bps)
- 特に改善: 商社 丸紅/三井物産 +242bps、電線 住電/古河 Sharpe 0.75→1.91
- 「全フィルタ」は絞り過ぎで N=132 まで減少 → 統計的有意性低下

### Phase 3d: 固定日数ホールド (統計的収束性の確認)
**最重要の発見**:
- Entry |Z|≥2 から 10日後、**80% の確率で Z は縮小**
- 20日後は 90%、40日後も 90%
- つまり「統計的平均回帰は確かに起きている」

にもかかわらず固定日数戦略は Sharpe ≈ 0:
- 勝ち (収束) は小さい、負け (発散) は巨大 (ファットテール)
- → **早期 MR exit + stop-loss + time exit** の 3 段構えが必要
- → **Phase 3a の設計を理論的に裏付ける結果**

## 最終推奨

### 採用
**#1 統合ペアトレード・ポートフォリオ** (18 ペア Equal-Weight)

**期待パフォーマンス**:
- 年率 Sharpe 1.37, t=4.55
- 年率リターン ~3% (レバ1倍、1ペアあたり資金1/18)
- レバ 3-5倍で年率 ~10-15% (Sharpe 維持、MDD ~1000-1300 bps)

### 運用注意点
1. **ポジションサイズ**: 各ペア 1/18 の等配分。β 正規化 (1+|β| でグロス換算)
2. **エントリー**: |Z|≥2 でスプレッド方向張り (ペアごとの最適 Zwin/MaxHold 使用)
3. **エグジット**: (a) |Z|<0.5 MR, (b) |Z|>4 Stop, (c) MaxHold 経過で Time
4. **再バランス**: β は trailing 60日 OLS で毎日更新
5. **コスト**: 8 bps/往復を見込み。分足では 16 bps と悪化するため日次推奨
6. **リバランスコスト**: 全ペア毎日回転ではないので実質低い (年間取引回数 70-80回/ペア × 18 = ~1400往復)

### 更なる改善候補 (OoS 検証必要)
1. **Regime+Vol フィルタの慎重な適用** — ペア別に効くもの/悪化するものを峻別
2. **Kelly サイジング** — 半減期の短いペアを overweight
3. **動的ペア選抜** — 直近2年の Sharpe でランキングし、上位 10 ペアのみ運用
4. **日中執行の改善** — 寄付1時間は避け、11:00-11:30 と 14:30-15:00 に発注
